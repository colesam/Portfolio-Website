YouTube Simple Viewer

Description

This project is a web application I built using ReactJS and the YouTube API. The application is a very simple interface for watching YouTube videos. Components load separately from one another resulting in a very fast and clean viewing experience.

This project was made by following along with Stephen Grider's React course on Udemy. This project differs from Stephen Grider's in styling and some parts of the implementation.

Features

  • Search for YouTube videos by clicking the Search button or hitting the 'Enter' key
  • Choose from a list of five YouTube videos
  • Continue searching for new YouTube videos while the current video remains on the page (implementation is specific to my project)

Code I Wrote

React Components

I wrote the index.js file and ReactJS components found in the root/src directory.

Inside index.js you will find the AJAX requests I wrote to the YouTube API. I used the Axios module to make the requests and run callbacks using its built in promises.

The files in root/src/components subdirectory contain primarily html in the form of JSX. The two components that I spent the most time on were root/src/components/video_list_item.js and root/src/components/video_detail.js.

SASS Styling

I styled this project using the SASS preprocessor. The SASS files and the generated CSS can both be found in the root/style directory.

I styled the UI for this app to be very similar to the way YouTube videos look because the interface is familiar to users. It has been simplified to only display search results instead of suggested videos. It also does not display any comments or description to keep the interface as clean and "Simple" as possible.

Difficulties

Setting Selected Video

The instructor for the course I was following along with would always initialize the selected video to be the first result returned from a video search. I wanted my application to be able to continue playing even if the videos in the video list were changing due to a search.

My first solution was to create two functions, one to populate the list of videos and one to load a specific video. I still wanted to use the first search result as the selected video when the page initializes, but on all subsequent searches I wanted the selected video to remain the same.

I quickly realized that I would need to use a promise in the constructor of my App class, because I would not be able to initialize the selected video until the first search had returned results.

My solution to this problem was to create a single function inside the App class in index.js.

videoSearch( searchTerm, callback )

This function made an AJAX request to YouTube using the searchTerm parameter, and then ran the callback parameter if it was provided. I simply provided the following as a callback to the first call to videoSearch():

() => { this.setState({ selectedVideo: this.state.videos[0] }); }

Including Description

I planned on including an expandable description for the video currently being played only to realize that the YouTube API returns these descriptions as a string. It's very common for YouTube video descriptions to contain formatted text including anchor tags and line breaks.

I considered using a regular expression to detect web addresses and convert them to anchor tags, but that would still leave me with a long block of text unseparated by line breaks. In the end, I decided to put a link to the YouTube video where the description would have been.
