Breathe Buddy

Description

This project is a web application I built with my partner, Mathew Schlichting, for our CISC 375 Web Development course at the University of St. Thomas.

The project was built using HTML5, CSS3, Bootstrap 4.0, and AngularJS. It combines the Google Maps API with the Open AQ API to create a web application that shows the user information about the air quality of a given location.

Features

  • Shows a map using the Google Maps API
  • Allows user to search by location name or lat/lng values
  • Binds map location to location input boxes to update location when user clicks and drags map
  • Allows user to select date within last 90 days to grab data from
  • Populates a table with air quality measurements based off of location and date
  • Shows a UI for filtering data by particle type or by minimum measurement per particle type
  • Pops up measurement data when map marker is hovered over
  • Clusters markers together when they occupy a small area on the map
  • Features custom made cluster PNG files to match site theme
  • Moves critical UI components inside the map on fullscreen mode
  • Allows for heatmap display when only one particle type is selected
  • Contains "About the Project" page with profile pictures of each partner and short bios

Code I Wrote

HTML and CSS

I did a majority of the html structure and styling for the project. All CSS files can be found in the root/css directory. The HTML is broken up into two files and incorporated into index.html by using ng-include directives. These two files can be found in the root/html directory.

Because the html for both pages are included in one html file, they are switched between using JQuery and CSS. I programmed this functionality in to allow for extremely fast loading between pages, and to allow the large png file from the about-the-project page to load while the user is on the app's page. This functionality is in root/js/app.js in the $(document).ready(fn) function.

DATE PICKER and JQUERY

I wrote the JavaScript that makes the custom date picker in the UI work. The reason I coded a custom date picker from scratch is because the Open AQ API only maintains data from the last 90 days. When you consider UX, it makes sense not to load any dates that are invalid options. As a result we use our custom datepicker that will only show you today's date and 90 days before it.

The code used to implement this can be found in root/js/controllers/airQualityController.js . You will want to look towards the bottom of the file for the section labeled "Date Picker".

Within the same file I wrote most of the code in the $scope.mapInit and the $scope.checkFullScreen functions in the "Basic Map Functions" sections to move UI html elements inside the map for fullscreen mode.

Difficulties

CSS ORGANIZATION

I tried my best to keep the CSS files as organized as possible but it still got a little messy. I broke my index.css file into many separate CSS files, each of which are also split apart by commented sections.

I believe part of the problem is that I vastly underestimated the amount of styling that would go into this project. If I could go back, I would use a SASS preprocessor to keep my CSS more organized.

NG-CLICK EVENTS

I had a lot of difficulty getting ng-click events to work properly in elements created by JQuery. The two types of elements giving me trouble were the month buttons and dates within the date picker.

I encountered these problems on the night before the deadline so I opted to use JQuery's .click( ) function because of its familiarity to me. Following AngularJS best practices is important to me but writing code that I understand and have confidence in is more important, especially in a time-crunch.
